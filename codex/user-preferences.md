# User Preferences

- The user wants precise, practical instructions.
- The user prefers not to lose context between sessions.
- The user wants clean, well-organized code.
- The user wants the work split into numbered phases `X/Y`.
- The user wants mobile compatibility.
- The user wants comments where useful, but not excessive comments.
- The user wants impressive design, but not over-engineered design.
- The user wants Hebrew RTL preserved.
- The user does not want unrelated files changed.
- The user expects exact reporting at the end of each phase.
